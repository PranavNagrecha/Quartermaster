Quartermaster

Issues your agent exactly the tools the mission needs — nothing more.

An offline, zero-dependency tool-router for MCP. No embedding model, no network, no API key.

GitHub → Quick start Benchmarks

The problem

Give a model 200 tools and every schema loads into context on every turn, and it must pick the right one from 200 lookalikes. Quartermaster funnels them down to a ranked shortlist first.

Funnel advises, model decides

  query
    │  tokenize → expand → BM25 (offline, no model)
    ▼
  top-8 shortlist + guidance ──► host LLM reads & picks ──► right tool

Why this one

  • Zero embedding model — a few hundred lines of dependency-free TypeScript.
  • Host-agnostic — any MCP client, any model, not just one API.
  • Advises, doesn't decide — returns a shortlist + guidance.
  • Offline & private — nothing phones home; air-gap friendly.

We don't claim best-in-class accuracy — pure lexical ranking trails hybrid embeddings at scale. The thesis under test: a zero-dependency hybrid (BM25 + offline expansion) good enough to be the no-model default. The benchmark says so in the open.

Install

npm install @pranavnpm/core

Status: early / alpha. Core ranker works (extracted from a production 170-tool router); proxy + plugin are scaffolded.

문제

도구가 200개면 매 턴마다 모든 스키마가 컨텍스트에 실리고, 비슷한 200개 중에서 골라야 합니다. Quartermaster는 먼저 순위가 매겨진 후보 목록으로 좁힙니다.

깔때기는 조언하고, 모델이 결정합니다

  질의 → 토큰화 → 확장 → BM25(오프라인) → 상위 8개 후보 → LLM이 선택
